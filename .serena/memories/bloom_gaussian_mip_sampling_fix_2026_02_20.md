Bloom gaussian compute blur had coordinate drift because higher mip horizontal passes used output dimensions for both output and input sampling. In BloomPass.writeBlurUniformsIfNeeded, level>0 dir=0 now writes inputSize as previous mip dimensions while outputSize stays current mip dimensions. Blur uniform expanded from 112 to 128 bytes to include inputSize and WGSL padding. In bloom.wgsl.ts createBloomBlurComputeShader, added inputSize to BlurComputeUniforms and mapOutputToInputCoord() to map output pixel centers to input pixel centers before textureLoad. This prevents bloom energy drifting toward bottom-right during downsample chain.
